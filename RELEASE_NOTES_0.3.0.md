# Shiju v0.3.0 — 输入体验升级

> 发布日期：2026-06-09
> 前序版本：v0.2.2

## 概述

v0.3.0 聚焦于输入体验的提升。标签候选从按钮行升级为带键盘导航的下拉列表，Markdown 工具条扩展到 9 个按钮并支持横向滚动，保存后自动定位到插入位置，并增加了 GitHub Actions CI 确保代码质量。

## 新增功能

### 🏷️ 标签候选 Dropdown

输入 `#` 后，候选标签以垂直下拉列表展示，支持：
- **匹配字符高亮** — 输入部分用 `<strong>` 标出
- **键盘导航** — ↑/↓ 选择，Enter 确认
- **鼠标交互** — 悬停高亮，点击选中

### 🔧 Markdown 工具条扩展

新增 4 个按钮，共 9 个：
- 斜体 `*text*`、行内代码 `` `code` ``、引用 `> quote`、分割线 `---`
- 移动端支持**横向滚动**，按钮不换行不压缩

### 📍 保存后自动定位

保存内容并打开今日日记后，编辑器自动**滚动到刚才插入的位置**，光标定位到该行开头。不再需要手动翻找。

### ⚙️ 设置页恢复默认

设置页底部新增「恢复默认设置」按钮，带确认对话框，一键恢复所有配置。

### 🔄 CI 自动化

新增 GitHub Actions 工作流，每次 push 到 main 或提交 PR 时自动运行 `npm ci && npm run build`，确保代码质量。
